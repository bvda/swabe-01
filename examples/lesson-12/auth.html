OAuth 2.0 Grant Flows

Endpoints

  • Implicit (ID Token) https://localhost:3000/oidc/auth?client_id=such_app&response_type=id_token&redirect_uri=https://127.0.0.1:3010/callback&nonce=foobar&scope=openid
  • Authorization Code https://localhost:3000/oidc/auth?client_id=code&response_type=code&redirect_uri=https://127.0.0.1:3010/callback&scope=openid offline_access
  • Authorization Code (Refresh Token) https://localhost:3000/oidc/auth?client_id=code&prompt=consent&response_type=code&redirect_uri=https://127.0.0.1:3010/callback&scope=openid offline_access
  • Authorization Code (PKCE) http://localhost:3000/auth?client_id=foo&response_type=code&redirect_uri=https://127.0.0.1:3010/callback&scope=openid&code_challenge=_drLS7o5FwkfUiBhlq2hwJnK_SC6yE7sKOde5O1fdzk&code_challenge_method=S256

Notes

If you're running Chrome and have trouble with the self-signed certificates, run chrome://flags/#allow-insecure-localhost in the address bar and enabled insecure localhost.
