Cloud Firestore

Cloud Firestore is a cloud-hosted, NoSQL database that your iOS, Android, and web apps can access directly via native SDKs

  • Flexiblitiy

    The Cloud Firestore data model supports flexible, hierarchical data structures

  • Expressive querying

    Use queries to retrieve individual, specific documents or to retrieve all the documents in a collection that match your query parameters

  • Realtime updates

    Cloud Firestore uses data synchronization to update data on any connected device

  • Offline support

    Cloud Firestore brings you the best of Google Cloud Platform's powerful infrastructure

  • Designed to scale

Document Database

Structuring data

Nested data in documents

  • Advantages

    Easy to set up and streamlines your data structure

  • Limitations

    Scalability can become an issue if your data expands over time

  • Possible use case

    In a chat app, for example, you might store a user's 3 most recently visited chat rooms as a nested list in their profile

Subcollections

  • Advantages

    The size of the parent document doesn't change. You also get full query capabilities on subcollections

  • Limitations

    You can't easily delete subcollections

  • Possible use case

    In the same chat app, for example, you might create collections of users or messages within chat room documents

Root-level collections

  • Advantages

    Root-level collections offer the most flexibility and scalability, along with powerful querying within each collection

  • Limitations

    Getting data that is naturally hierarchical might become increasingly complex as your database grows

  • Possible use case

    In the same chat app, for example, you might create one collection for users and another for rooms and messages

Cloud Functions

Cloud Functions

  • Serverless Backend

    Run all your backend code in the cloud

  • Triggers

    Invoke functions based on events from other Firebase products

  • Security

    Run application logic in a controlled environment

  • Zero maintenance

    Server provisioning, resource scaling and configuration is handled

Notify

Notify users when something interesting happens

  • The function triggers on writes to the Realtime Database path where followers are stored.

  • The function composes a message to send via FCM.

  • FCM sends the notification message to the user's device.

Sanitization

Perform database sanitization and maintenance

  • The function's database event handler listens for write events on a specific path, and retrieves event data containing the text of any chat messages.

  • The function processes the text to detect and scrub any inappropriate language.

  • The function writes the updated text back to the database.

Execute intensive tasks

Execute intensive tasks in the cloud instead of in your app

  • A function triggers when an image file is uploaded to Storage.

  • The function downloads the image and creates a thumbnail version of it.

  • The function writes that thumbnail location to the database, so a client app can find and use it.

  • The function uploads the thumbnail back to Storage in a new location.

  • The app downloads the thumbnail link.

Integration

Integrate with third-party services and APIs

  • A user pushes commits to a GitHub repo.

  • An HTTPS function triggers via the GitHub webhook API.

  • The function sends a notification of the commit to a team Slack channel.
